# Summary: Dijkstra on the Foolishness of Natural Language Programming

## Main Arguments

**Dijkstra's Core Thesis**: Natural language programming is fundamentally flawed because it attempts to solve the wrong problem. The precision required in formal programming languages isn't a bug—it's a feature that enables reliable computation.

## Key Historical Observations

**Evolution of Programming Languages**:
- Machine code was rightfully identified as too risky due to lack of redundancy
- High-level languages improved error detection and prevention
- However, some programmers actually preferred undetected errors over error messages they had to address

**Mathematical Precedent**:
- Greek mathematics stagnated due to reliance on verbal/pictorial methods
- Muslim algebra declined when it returned to rhetoric over symbolic notation
- Modern mathematics only advanced through formal symbolism (contributions by Vieta, Descartes, Leibniz, Boole)

## The Interface Problem

**Dijkstra's Warning**: Changing interfaces doesn't simply redistribute work—it often increases complexity on both sides. Natural language interfaces:
- Dramatically increase the machine's computational burden
- Don't necessarily simplify the human experience
- Create communication overhead that adds to total system complexity

## Modern Relevance (Commentator's Insights)

**Current AI/LLM Context**:
- English's imprecision causes misinterpretation in AI coding tools
- Emergence of "pseudo-languages" designed for LLMs
- Massive computational resources (nuclear power plants, thousands of gallons of water) now dedicated to natural language processing

## Key Takeaways

1. **Precision is Valuable**: Formal programming languages' requirement for accuracy isn't a limitation to overcome but a necessary tool for reliable computation

2. **Interface Changes Have Hidden Costs**: Adding abstraction layers often increases total system complexity rather than reducing it

3. **Historical Pattern**: Every major advancement in mathematics and computation has come through embracing formal symbolism, not avoiding it

4. **Resource Reality**: Natural language processing requires enormous computational resources, validating Dijkstra's prediction about increased machine burden

## Actionable Insight

Rather than seeking to eliminate formal programming concepts, developers should focus on better tooling and education that makes formal methods more accessible while preserving their precision and reliability.
